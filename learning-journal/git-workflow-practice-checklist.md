# Git Workflow Practice Checklist

## Week 1 Day 2 Practice Summary

This checklist summarizes the Git workflow tasks practiced during Week 1 Day 2.

## Commands Practiced

* `git status`
* `git add`
* `git commit`
* `git branch`
* `git checkout`
* `git pull`
* `git push`
* `git merge`

## Workflow Practiced

1. Checked the current repository status.

2. Created a separate branch for Day 2 work.

3. Added learning journal updates.

4. Staged specific files for clean commits.

5. Created meaningful commits with clear messages.

6. Prepared the branch for pushing to GitHub.

7. Prepared for Pull Request creation and merge workflow.

## Key Learning Points

* Branches help keep work separate from the main branch.
* Commits act as saved checkpoints in project history.
* Meaningful commit messages make the repository easier to review.
* Pull Requests help review changes before merging.
* Merge conflicts need to be resolved carefully by checking the affected file manually.

## My Understanding

Git workflow is not only about running commands. It is about keeping work organized, reviewable, and safe. A clean branch and meaningful commits make it easier for mentors or team members to understand what was done.
